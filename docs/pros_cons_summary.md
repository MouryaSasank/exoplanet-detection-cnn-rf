# Pros and Cons Summary: Hybrid CNN+RF Exoplanet Detection Pipeline

**Project:** CNN-Based Feature Learning Combined with Random Forest for Robust Exoplanet Classification Using NASA Kepler Light Curves  
**Date:** April 2026  
**Author:** Mourya Sasank

## 1. Architectural Strengths

The pipeline adopts a compact three-block 1D convolutional architecture with a deliberate filter progression of 32→64→128, which is highly appropriate for a dataset containing only 37 confirmed exoplanet candidates. This conservative sizing prevents the network from memorising the limited positive examples — a failure mode that larger architectures such as ResNet- or Inception-style blocks would almost certainly exhibit at this data scale. The use of GlobalAveragePooling1D in place of a Flatten layer is particularly significant: where a Flatten operation would produce a fully-connected junction of approximately 26 million parameters (depending on the final temporal dimension), GlobalAveragePooling1D collapses the 128-filter feature map into a 128-dimensional vector, feeding into a 64-unit dense layer that yields roughly 65 thousand trainable parameters in total. This three-order-of-magnitude reduction in parameter count fundamentally changes the overfitting dynamics of the model and renders training feasible on a dataset of this size without aggressive data augmentation.

The choice of Focal Loss over standard binary cross-entropy represents a well-justified departure from conventional practice. In a dataset where non-planet examples outnumber confirmed planets by roughly 100:1, a model trained with standard BCE rapidly learns to predict the majority class exclusively, achieving 99% accuracy with zero recall. Focal Loss addresses this by down-weighting well-classified examples through the modulating factor (1 − p_t)^γ, concentrating the gradient signal on the informative hard examples. The configuration of γ = 2.0 and α = 0.75 provides a balanced trade-off that has been shown to be effective in comparably skewed classification settings. L2 regularisation applied to all convolutional kernels, with a coefficient of 0.001, provides a complementary defence against overfitting by constraining the magnitude of learned weights, discouraging the network from placing excessive reliance on any single spectral feature.

## 2. Evaluation Design Strengths

The selection of Precision-Recall Area Under Curve (PR-AUC) as the primary evaluation metric, rather than the more common ROC-AUC, demonstrates a mature understanding of imbalanced classification. ROC-AUC can return deceptively optimistic values when the number of true negatives vastly exceeds false positives; in contrast, PR-AUC is dominated by the model's ability to correctly identify the rare positive class, making it an intrinsically harder and more informative metric for this problem domain. By reporting PR-AUC alongside F1-score and Matthews Correlation Coefficient, the pipeline avoids the common trap of relying on a single number that can be gamed by threshold manipulation.

The dual SMOTE strategy represents a carefully reasoned approach to managing class imbalance at two different stages of the pipeline. For the CNN, SMOTE is deliberately not applied to the raw flux data; instead, class weights handle the imbalance during training, because generating synthetic light curves through linear interpolation of raw flux values would produce physically implausible transit signals that do not correspond to any real astrophysical phenomenon. SMOTE is applied only at the hybrid stage, where it operates on the 64-dimensional learned feature space extracted from the CNN's penultimate dense layer, and at the baseline stage, where it operates on normalised flux features. Critically, in both cases SMOTE is applied strictly after the train-test split, eliminating any risk of data leakage from synthetic examples contaminating the evaluation set.

The adaptive threshold search mechanism, combined with a three-tier fallback guarantee, ensures that the pipeline always produces valid predictions even under adversarial conditions. The primary search sweeps a fine-grained grid of thresholds via `np.linspace` to maximise F1-score; if this search returns F1 = 0 (indicating that no threshold produced any positive predictions), the system falls back to a top-1% percentile threshold, which mechanically guarantees that at least 1% of samples are classified as positive. If this too fails, a final top-0.5% percentile is attempted. This three-tier structure means the pipeline can never enter a state where it produces all-negative predictions, which would render downstream evaluation metrics undefined.

## 3. Acknowledged Limitations

The most fundamental limitation of this study is the extremely small positive test set, which contains approximately five confirmed exoplanet candidates. At this sample size, the addition or removal of a single true positive from the prediction set can shift the test F1-score by 0.15–0.20 and the precision by 0.10–0.25, rendering test-set metrics inherently unstable. While the cross-validated PR-AUC provides a more robust aggregate estimate by averaging across multiple folds, the headline test metrics are unavoidably noisy and should be interpreted with caution. This is not a modelling failure but rather a consequence of the Kepler dataset's extreme rarity of confirmed exoplanets — a constraint that no architectural choice can fully overcome.

Threshold-based metric instability compounds the small-sample problem. Because the pipeline derives its classification decision from a continuous probability score and a learned threshold, small perturbations in the probability distribution — arising from random weight initialisation, batch ordering, or stochastic dropout — can cause the optimal threshold to shift by several hundredths. When the positive class contains only five members in the test set, such threshold shifts routinely toggle one or two predictions, producing the observed variance in per-run metrics. The bootstrapped confidence intervals partially address this, but they reflect resampling variance within a single run rather than cross-run initialisation variance.

The pipeline has been validated exclusively on NASA Kepler Quarter 3 light curves. No cross-mission validation has been performed against TESS (Transiting Exoplanet Survey Satellite) or K2 (Kepler's extended mission) data, which differ from Kepler in cadence, noise characteristics, systematic artefacts, and observation baseline. It is therefore unknown whether the learned CNN features transfer to these instruments. Transfer learning experiments using TESS light curves would be required to establish the generalisability of the approach, and their absence limits the external validity of the current results.

Finally, the 64-dimensional latent feature vector extracted by the CNN, while demonstrably useful for downstream Random Forest classification, is not directly interpretable by astronomers. Unlike handcrafted features such as transit depth, duration, or orbital period, the learned dimensions do not correspond to known astrophysical quantities. The SHAP-based explainability analysis partially mitigates this by identifying which latent dimensions contribute most to individual predictions, but it does not resolve the underlying opacity of the feature space. Domain experts reviewing the pipeline's predictions must ultimately trust the aggregate evaluation metrics rather than being able to inspect the physical reasoning behind individual classifications.

## 4. Risk Mitigation Already Implemented

To counteract the instability arising from the small positive test set, the pipeline implements bootstrapped confidence intervals (n = 1000 iterations) for all key metrics. Each iteration resamples the test predictions with replacement and recomputes precision, recall, F1-score, and PR-AUC, yielding a 95% confidence interval that quantifies the range of plausible values given the sampling noise. This approach reveals whether an observed F1-score of, say, 0.80 truly reflects model performance or merely a fortunate alignment of test examples in a single split. The bootstrap intervals are persisted to disk alongside the point estimates, ensuring that any downstream consumer of the results can assess uncertainty without re-running the pipeline.

The cross-validated mean PR-AUC serves as the pipeline's primary trust metric, deliberately chosen over single-split test metrics. By averaging PR-AUC across five stratified folds, each containing a different allocation of the rare positive examples, the CV estimate smooths out the volatility that plagues any single train-test partition. When the CV PR-AUC and the test PR-AUC are consistent (within the bootstrap interval), this provides meaningful evidence that the model has learned a genuine signal rather than overfitting to a particular split. When they diverge, it signals instability that warrants investigation.

The percentile fallback guarantee in the baseline model's threshold search provides a mechanical safety net against total prediction failure. In practice, probability calibration (particularly sigmoid-based Platt scaling) can compress the predicted probability range into a narrow band far below the default 0.5 decision boundary, causing a naïve threshold search to find zero positives. The three-tier fallback — linspace search, then top-1% percentile, then top-0.5% percentile — ensures that even under the most adversarial calibration artefact, the baseline always produces a non-degenerate set of predictions. This guarantee was added after observing that CalibratedClassifierCV with the sigmoid method compressed baseline probabilities into the 0.001–0.07 range, a failure mode that is now fully handled.
